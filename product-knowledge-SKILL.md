# skills/product-knowledge-SKILL.md

# 技能名称
Product Knowledge Translator

# 目标
把产品信息翻译成用户能理解、愿意传播、可用于转化的表达。

# 输入
- sync/products.md
- sync/asset_index.md
- 当前任务说明

# 核心原则
- 不做参数堆砌
- 不先写技术名词
- 不脱离用户场景
- 默认把产品放回“Human × Car × Home”或更大生态关系中理解
- 如果产品天然适合单点表达，也要说明它在整体体验中的位置

# 处理流程
1. 提取当前重点产品
2. 提取：
   - 核心卖点
   - 关键体验收益
   - 适用人群
   - 典型场景
   - 常见疑问
   - 是否与其他设备/服务形成生态协同
3. 将产品信息重写为以下层级：
   - 一句话价值
   - 三条用户收益
   - 一条生态关系说明
   - 一条可信证据或素材建议
4. 如果资料不足，不得虚构
5. 若存在多个产品，先判断主副关系，不平均分配篇幅

# 默认输出模板
## 产品理解卡
- 产品名称：
- 一句话价值：
- 适合谁：
- 典型场景：
- 三个关键收益：
- 生态协同关系：
- 推荐内容方向：
- 可调用素材：
- 风险提醒：

# 特别规则
当产品涉及高端化或旗舰表达时：
- 强调体验成立、技术兑现、生态完整
- 避免单纯“奢侈化”或空洞“高端感”
